1//! # Minimal Unsafe C API for OpenMP Parser
2//!
3//! This module provides a direct pointer-based C FFI with minimal unsafe code.
4//!
5//! ## Design Philosophy: Direct Pointers vs Handles
6//!
7//! We use **raw C pointers** to Rust structs instead of opaque handles for:
8//! - **Idiomatic C API**: Standard malloc/free pattern familiar to C programmers
9//! - **Simple memory model**: No global registry, no handle bookkeeping
10//! - **Easy integration**: Works naturally with C/C++ code
11//! - **Minimal code**: 632 lines vs 4000+ lines of handle management
12//!
13//! ## Safety Analysis: 18 Unsafe Blocks (~60 lines)
14//!
15//! All unsafe blocks are:
16//! - **NULL-checked** before dereferencing
17//! - **Documented** with explicit safety requirements
18//! - **Isolated** only at FFI boundary, never in business logic
19//! - **Auditable**: ~0.9% of file (60 unsafe lines / 632 total)
20//!
21//! ## Case-Insensitive Matching: String Allocation Tradeoff
22//!
23//! Functions `directive_name_to_kind()` and `convert_clause()` allocate a String
24//! for case-insensitive matching (Fortran uses uppercase, C uses lowercase).
25//!
26//! **Why not optimize with `eq_ignore_ascii_case()`?**
27//! - Constants generator (`src/constants_gen.rs`) requires `match` expressions
28//! - Parser uses syn crate to extract directive/clause mappings from AST
29//! - Cannot parse if-else chains → must use `match normalized_name.as_str()`
30//! - String allocation is necessary for match arm patterns
31//!
32//! **Is this a performance issue?**
33//! - No: These functions are called once per directive/clause at API boundary
34//! - Typical usage: Parse a few dozen directives in an entire program
35//! - String allocation cost is negligible compared to parsing overhead
36//!
37//! **Future optimization path**: Update `constants_gen.rs` to parse if-else chains,
38//! then use `eq_ignore_ascii_case()` without allocations.
39//!
40//! ## Learning Rust: Why Unsafe is Needed at FFI Boundary
41//!
42//! 1. **C strings** → Rust strings: `CStr::from_ptr()` requires unsafe
43//! 2. **Memory ownership** transfer: `Box::into_raw()` / `Box::from_raw()`
44//! 3. **Raw pointer dereferencing**: C passes pointers, we must dereference
45//!
46//! Safe Rust cannot verify C's guarantees, so we explicitly document them.
47//!
48//! ## C Caller Responsibilities
49//!
50//! C callers MUST:
51//! - ✅ Check for NULL returns before use
52//! - ✅ Call `_free()` functions to prevent memory leaks
53//! - ✅ Never use pointers after calling `_free()`
54//! - ✅ Pass only valid null-terminated strings
55//! - ✅ Not modify strings returned by this API
56

144/// Clause-specific data stored in a C union
145///
146/// Learning Rust: Why ManuallyDrop?
147/// =================================
148/// Unions in Rust don't know which variant is active, so they can't
149/// automatically call destructors. ManuallyDrop prevents automatic drops
150/// and lets us manually free resources when we know which variant is active.
151#[repr(C)]
152union ClauseData {
153    schedule: ManuallyDrop<ScheduleData>,
154    reduction: ManuallyDrop<ReductionData>,
155    default: i32,
156    variables: *mut OmpStringList,
157}

164}

195/// Parse an OpenMP directive from a C string.
196///
197/// ## Parameters
198/// - `input`: Null-terminated C string containing the directive
199///
200/// ## Returns
201/// - Pointer to `OmpDirective` on success
202/// - NULL on parse failure or NULL input
203///
204/// ## Safety
205/// Caller must:
206/// - Pass valid null-terminated C string or NULL
207/// - Call `roup_directive_free()` on the returned pointer
208///
209/// ## Example
210/// ```c
211/// OmpDirective* dir = roup_parse("#pragma omp parallel");
212/// if (dir) {
213///     // use directive
214///     roup_directive_free(dir);
215/// }
216/// ```
217#[no_mangle]
218pub extern "C" fn roup_parse(input: *const c_char) -> *mut OmpDirective {
219    // NULL check
220    if input.is_null() {
221        return ptr::null_mut();
222    }
223
224    // UNSAFE BLOCK 1: Convert C string to Rust &str
225    // Safety: Caller guarantees valid null-terminated C string
226    let c_str = unsafe { CStr::from_ptr(input) };
227
228    let rust_str = match c_str.to_str() {
229        Ok(s) => s,
230        Err(_) => return ptr::null_mut(), // Invalid UTF-8
231    };
232
233    // Parse using safe Rust parser
234    let directive = match parse_omp_directive(rust_str) {
235        Ok((_, dir)) => dir,
236        Err(_) => return ptr::null_mut(), // Parse error
237    };
238
239    // Convert to C-compatible format
240    let c_directive = OmpDirective {
241        name: allocate_c_string(directive.name.as_ref()),
242        clauses: directive
243            .clauses
244            .into_iter()
245            .map(|c| convert_clause(&c))
246            .collect(),
247    };
248
249    // UNSAFE BLOCK 2: Convert Box to raw pointer for C
250    // Safety: Caller will call roup_directive_free() to deallocate
251    Box::into_raw(Box::new(c_directive))
252}
253
254/// Free a directive allocated by `roup_parse()`.
255///
256/// ## Safety
257/// - Must only be called once per directive
258/// - Pointer must be from `roup_parse()`
259/// - Do not use pointer after calling this function
260#[no_mangle]
261pub extern "C" fn roup_directive_free(directive: *mut OmpDirective) {
262    if directive.is_null() {
263        return;
264    }
265
266    // UNSAFE BLOCK 3: Convert raw pointer back to Box for deallocation
267    // Safety: Pointer came from Box::into_raw in roup_parse
268    unsafe {
269        let boxed = Box::from_raw(directive);
270
271        // Free the name string (was allocated with CString::into_raw)
272        if !boxed.name.is_null() {
273            drop(CString::from_raw(boxed.name as *mut c_char));
274        }
275
276        // Free clause data
277        for clause in &boxed.clauses {
278            free_clause_data(clause);
279        }
280
281        // Box is dropped here, freeing memory
282    }
283}

285/// Parse an OpenMP directive with explicit language specification.
286///
287/// ## Parameters
288/// - `input`: Null-terminated string containing the directive
289/// - `language`: Language format (ROUP_LANG_C, ROUP_LANG_FORTRAN_FREE, ROUP_LANG_FORTRAN_FIXED)
290///
291/// ## Returns
292/// - Pointer to `OmpDirective` on success
293/// - `NULL` on error:
294///   - `input` is NULL
295///   - `language` is not a valid ROUP_LANG_* constant
296///   - `input` contains invalid UTF-8
297///   - Parse error (invalid OpenMP directive syntax)
298///
299/// ## Error Handling
300/// This function returns `NULL` for all error conditions without detailed error codes.
301/// There is no way to distinguish between different error types (invalid language,
302/// NULL input, UTF-8 error, or parse failure) from the return value alone.
303///
304/// Callers should:
305/// - Validate `language` parameter before calling (use only ROUP_LANG_* constants)
306/// - Ensure `input` is non-NULL and valid UTF-8
307/// - Verify directive syntax is correct
308/// - For debugging, enable logging or use a separate validation layer
309///
310/// For a version with detailed error codes, consider using the Rust API directly.
311///
312/// ## Example (Fortran free-form)
313/// ```c
314/// OmpDirective* dir = roup_parse_with_language("!$OMP PARALLEL PRIVATE(A)", ROUP_LANG_FORTRAN_FREE);
315/// if (dir) {
316///     // Use directive
317///     roup_directive_free(dir);
318/// } else {
319///     // Handle error: NULL, invalid language, invalid UTF-8, or parse failure
320///     fprintf(stderr, "Failed to parse directive\n");
321/// }
322/// ```
323#[no_mangle]
324pub extern "C" fn roup_parse_with_language(
325    input: *const c_char,
326    language: i32,
327) -> *mut OmpDirective {
328    // NULL check
329    if input.is_null() {
330        return ptr::null_mut();
331    }
332
333    // Convert language code to Language enum using explicit constants
334    // Return NULL for invalid language values
335    let lang = match language {
336        ROUP_LANG_C => Language::C,
337        ROUP_LANG_FORTRAN_FREE => Language::FortranFree,
338        ROUP_LANG_FORTRAN_FIXED => Language::FortranFixed,
339        _ => return ptr::null_mut(), // Invalid language value
340    };
341
342    // UNSAFE BLOCK: Convert C string to Rust &str
343    let c_str = unsafe { CStr::from_ptr(input) };
344
345    let rust_str = match c_str.to_str() {
346        Ok(s) => s,
347        Err(_) => return ptr::null_mut(),
348    };
349
350    // Create parser with specified language
351    let parser = openmp::parser().with_language(lang);
352
353    // Parse using language-aware parser
354    let directive = match parser.parse(rust_str) {
355        Ok((_, dir)) => dir,
356        Err(_) => return ptr::null_mut(),
357    };
358
359    // Convert to C-compatible format
360    let c_directive = OmpDirective {
361        name: allocate_c_string(directive.name.as_ref()),
362        clauses: directive
363            .clauses
364            .into_iter()
365            .map(|c| convert_clause(&c))
366            .collect(),
367    };
368
369    Box::into_raw(Box::new(c_directive))
370}

376/// Convert an OpenMP directive from one language to another.
377///
378/// This function translates OpenMP directives between C/C++ and Fortran syntax:
379/// - Sentinel conversion: `#pragma omp` ↔ `!$omp`
380/// - Loop directive names: `for` ↔ `do`, `parallel for` ↔ `parallel do`, etc.
381/// - Clause preservation: All clauses are preserved as-is
382///
383/// ## Parameters
384/// - `input`: Null-terminated string containing the directive
385/// - `from_language`: Source language (ROUP_LANG_C, ROUP_LANG_FORTRAN_FREE, ROUP_LANG_FORTRAN_FIXED)
386/// - `to_language`: Target language (ROUP_LANG_C, ROUP_LANG_FORTRAN_FREE)
387///
388/// ## Returns
389/// - Pointer to null-terminated C string with translated directive on success
390/// - `NULL` on error:
391///   - `input` is NULL
392///   - `from_language` or `to_language` is invalid
393///   - `input` contains invalid UTF-8
394///   - Parse error (invalid OpenMP directive syntax)
395///
396/// ## Memory Management
397/// The returned string is heap-allocated and must be freed by calling `roup_string_free()`.
398///
399/// ## Limitations
400/// - **Expression translation**: Expressions within clauses are NOT translated
401///   (e.g., C syntax like `arr[i]` remains unchanged)
402/// - **Fixed-form output**: Only free-form `!$omp` output is supported (ROUP_LANG_FORTRAN_FIXED not supported as target)
403/// - **Surrounding code**: Only directive lines are translated, not actual source code
404///
405/// ## Example (C to Fortran)
406/// ```c
407/// const char* c_input = "#pragma omp parallel for private(i) schedule(static, 4)";
408/// char* fortran = roup_convert_language(c_input, ROUP_LANG_C, ROUP_LANG_FORTRAN_FREE);
409/// if (fortran) {
410///     printf("%s\n", fortran);  // Output: !$omp parallel do private(i) schedule(static, 4)
411///     roup_string_free(fortran);
412/// }
413/// ```
414///
415/// ## Example (Fortran to C)
416/// ```c
417/// const char* fortran_input = "!$omp parallel do private(i)";
418/// char* c_output = roup_convert_language(fortran_input, ROUP_LANG_FORTRAN_FREE, ROUP_LANG_C);
419/// if (c_output) {
420///     printf("%s\n", c_output);  // Output: #pragma omp parallel for private(i)
421///     roup_string_free(c_output);
422/// }
423/// ```
424#[no_mangle]
425pub extern "C" fn roup_convert_language(
426    input: *const c_char,
427    from_language: i32,
428    to_language: i32,
429) -> *mut c_char {
430    // NULL check
431    if input.is_null() {
432        return ptr::null_mut();
433    }
434
435    // Convert language codes to IR Language enum
436    let from_lang = match language_code_to_ir_language(from_language) {
437        Some(lang) => lang,
438        None => return ptr::null_mut(), // Invalid from_language
439    };
440
441    let to_lang = match language_code_to_ir_language(to_language) {
442        Some(lang) => lang,
443        None => return ptr::null_mut(), // Invalid to_language
444    };
445
446    // Convert C string to Rust &str
447    let c_str = unsafe { CStr::from_ptr(input) };
448
449    let rust_str = match c_str.to_str() {
450        Ok(s) => s,
451        Err(_) => return ptr::null_mut(), // Invalid UTF-8
452    };
453
454    // Return early if input is empty
455    if rust_str.trim().is_empty() {
456        return ptr::null_mut();
457    }
458
459    // Convert from_language code to lexer::Language for parser
460    // IMPORTANT: Map language code directly to preserve fixed-form vs free-form distinction
461    let lexer_lang = match from_language {
462        ROUP_LANG_C => Language::C,
463        ROUP_LANG_FORTRAN_FREE => Language::FortranFree,
464        ROUP_LANG_FORTRAN_FIXED => Language::FortranFixed,
465        _ => return ptr::null_mut(), // Invalid from_language
466    };
467
468    // Parse the directive with the source language
469    let parser = openmp::parser().with_language(lexer_lang);
470    let (rest, directive) = match parser.parse(rust_str) {
471        Ok(result) => result,
472        Err(_) => return ptr::null_mut(), // Parse error
473    };
474
475    // Check for unparsed trailing input
476    if !rest.trim().is_empty() {
477        return ptr::null_mut();
478    }
479
480    // Convert to IR with source language context
481    let config = ParserConfig::with_parsing(from_lang);
482    let ir = match convert_directive(&directive, SourceLocation::start(), from_lang, &config) {
483        Ok(ir) => ir,
484        Err(_) => return ptr::null_mut(), // Conversion error
485    };
486
487    // Translate to target language
488    // Note: We don't modify the IR's language field, just render in target language
489    let output_str = ir.to_string_for_language(to_lang);
490
491    // Allocate C string for return
492    match CString::new(output_str) {
493        Ok(c_string) => c_string.into_raw(),
494        Err(_) => ptr::null_mut(), // String contains null bytes (shouldn't happen)
495    }
496}

498/// Free a string allocated by `roup_convert_language()`.
499///
500/// ## Parameters
501/// - `ptr`: Pointer to string returned by `roup_convert_language()`
502///
503/// ## Safety
504/// - Must only be called once per string
505/// - Pointer must be from `roup_convert_language()`
506/// - Do not use pointer after calling this function
507///
508/// ## Example
509/// ```c
510/// char* output = roup_convert_language(input, ROUP_LANG_C, ROUP_LANG_FORTRAN_FREE);
511/// if (output) {
512///     printf("%s\n", output);
513///     roup_string_free(output);
514/// }
515/// ```
516#[no_mangle]
517pub extern "C" fn roup_string_free(ptr: *mut c_char) {
518    if ptr.is_null() {
519        return;
520    }
521
522    // UNSAFE: Convert raw pointer back to CString for deallocation
523    // Safety: Pointer came from CString::into_raw in roup_convert_language
524    unsafe {
525        drop(CString::from_raw(ptr));
526    }
527}

546/// Get directive kind.
547///
548/// Returns -1 if directive is NULL.
549#[no_mangle]
550pub extern "C" fn roup_directive_kind(directive: *const OmpDirective) -> i32 {
551    if directive.is_null() {
552        return -1;
553    }
554
555    // UNSAFE BLOCK 4: Dereference pointer
556    // Safety: Caller guarantees valid pointer from roup_parse
557    unsafe {
558        let dir = &*directive;
559
560        // Use the canonical lookup to map the stored directive name into
561        // a `DirectiveName` enum, then map that enum to the C integer code.
562        if dir.name.is_null() {
563            return -1;
564        }
565
566        let c_str = CStr::from_ptr(dir.name);
567        let name_str = match c_str.to_str() {
568            Ok(s) => s,
569            Err(_) => return -1,
570        };
571
572        let dname = lookup_directive_name(name_str);
573        directive_name_enum_to_kind(dname)
574    }
575}

637/// Get next clause from iterator.
638///
639/// ## Parameters
640/// - `iter`: Iterator from `roup_directive_clauses_iter()`
641/// - `out`: Output pointer to write clause pointer
642///
643/// ## Returns
644/// - 1 if clause available (written to `out`)
645/// - 0 if no more clauses or NULL inputs
646#[no_mangle]
647pub extern "C" fn roup_clause_iterator_next(
648    iter: *mut OmpClauseIterator,
649    out: *mut *const OmpClause,
650) -> i32 {
651    // NULL checks
652    if iter.is_null() || out.is_null() {
653        return 0;
654    }
655
656    // UNSAFE BLOCK 5: Dereference iterator
657    // Safety: Caller guarantees valid iterator pointer
658    unsafe {
659        let iterator = &mut *iter;
660
661        if iterator.index >= iterator.clauses.len() {
662            return 0; // No more items
663        }
664
665        let clause_ptr = iterator.clauses[iterator.index];
666        iterator.index += 1;
667
668        // UNSAFE BLOCK 6: Write to output pointer
669        // Safety: Caller guarantees valid output pointer
670        *out = clause_ptr;
671        1
672    }
673}

769/// Get variable list from clause (private, shared, reduction, etc.).
770///
771/// Returns NULL if clause is NULL or has no variables.
772/// Caller must call `roup_string_list_free()`.

775    if clause.is_null() {
776        return ptr::null_mut();
777    }

779    // UNSAFE BLOCK 8: Dereference clause for variable check
780    // Safety: Caller guarantees valid clause pointer
781    unsafe {

784        // Check if this clause type has variables
785        // Kinds 2-5 are private/shared/firstprivate/lastprivate
786        if c.kind < 2 || c.kind > 6 {
787            return ptr::null_mut();
788        }

790        // For now, return empty list (would need clause parsing enhancement)
791        let list = OmpStringList { items: Vec::new() };
792        Box::into_raw(Box::new(list))
793    }
794}

811/// Get string at index from list.
812///
813/// Returns NULL if list is NULL or index out of bounds.
814/// Returned pointer is valid until list is freed.
815#[no_mangle]
816pub extern "C" fn roup_string_list_get(list: *const OmpStringList, index: i32) -> *const c_char {
817    if list.is_null() || index < 0 {
818        return ptr::null();
819    }
820
821    // UNSAFE BLOCK 9: Dereference list
822    // Safety: Caller guarantees valid list pointer
823    unsafe {
824        let l = &*list;
825        let idx = index as usize;
826
827        if idx >= l.items.len() {
828            return ptr::null();
829        }
830
831        l.items[idx]
832    }
833}

835/// Free string list.
836#[no_mangle]
837pub extern "C" fn roup_string_list_free(list: *mut OmpStringList) {
838    if list.is_null() {
839        return;
840    }
841
842    // UNSAFE BLOCK 10: Free list and strings
843    // Safety: Pointer from roup_clause_variables
844    unsafe {
845        let boxed = Box::from_raw(list);
846
847        // Free each C string (was allocated with CString::into_raw)
848        for item_ptr in &boxed.items {
849            if !item_ptr.is_null() {
850                drop(CString::from_raw(*item_ptr as *mut c_char));
851            }
852        }
853
854        // Box dropped here
855    }
856}

865/// Convert language code to IR Language enum.
866///
867/// Maps the C API language constants to the ir::Language enum used for
868/// semantic representation and translation.
869///
870/// ## Language Code Mapping:
871/// - 0 (ROUP_LANG_C) → Language::C (use this for both C and C++)
872/// - 1 (ROUP_LANG_FORTRAN_FREE) → Language::Fortran
873/// - 2 (ROUP_LANG_FORTRAN_FIXED) → Language::Fortran
874///
875/// Note: Both Fortran variants map to Language::Fortran in IR, as the
876/// distinction between free-form and fixed-form is a lexical concern,
877/// not a semantic one. There is no separate constant for C++; use ROUP_LANG_C for both C and C++.
878fn language_code_to_ir_language(code: i32) -> Option<IrLanguage> {
879    match code {
880        ROUP_LANG_C => Some(IrLanguage::C), // C/C++ use same OpenMP syntax
881        ROUP_LANG_FORTRAN_FREE => Some(IrLanguage::Fortran),
882        ROUP_LANG_FORTRAN_FIXED => Some(IrLanguage::Fortran),
883        _ => None, // Invalid language code
884    }

887/// Allocate a C string from Rust &str.
888///
889/// Creates a heap-allocated, null-terminated C string from Rust string.
890/// The returned pointer must be freed with CString::from_raw() later.
891///
892/// ## How it works:
893/// 1. CString::new() creates null-terminated copy
894/// 2. into_raw() gives us ownership of the pointer
895/// 3. Caller (or roup_directive_free) must eventually free it
896fn allocate_c_string(s: &str) -> *const c_char {
897    let c_string = std::ffi::CString::new(s).unwrap();
898    c_string.into_raw() as *const c_char
899}

901/// Find a top-level colon (:) not nested inside parentheses. Returns (left, right)
902fn split_once_top_level_colon(input: &str) -> Option<(&str, &str)> {
903    let mut depth: isize = 0;
904    for (i, ch) in input.char_indices() {
905        match ch {
906            '(' => depth += 1,
907            ')' => depth -= 1,
908            ':' => {
909                if depth == 0 {
910                    let left = &input[..i];
911                    let right = &input[i + 1..];
912                    return Some((left, right));
913                }
914            }
915            _ => {}
916        }
917    }
918    None
919}

921/// Convert Rust Clause to C-compatible OmpClause.
922///
923/// Maps clause names to integer kind codes (C doesn't have Rust enums).
924/// Each clause type gets a unique ID and appropriate data representation.
925///
926/// ## Clause Kind Mapping:
927/// - 0 = num_threads    - 6 = reduction
928/// - 1 = if             - 7 = schedule
929/// - 2 = private        - 8 = collapse
930/// - 3 = shared         - 9 = ordered
931/// - 4 = firstprivate   - 10 = nowait
932/// - 5 = lastprivate    - 11 = default
933/// - 999 = unknown
934fn convert_clause(clause: &Clause) -> OmpClause {
935    // Normalize clause name to lowercase for case-insensitive matching
936    // (Fortran clauses are uppercase, C clauses are lowercase)
937    // Note: One String allocation per clause is acceptable at C API boundary.
938    // Alternative (build-time constant map) requires updating constants_gen.rs
939    // to parse if-else chains instead of match expressions.
940    let normalized_name = clause.name.to_ascii_lowercase();
941
942    let clause_enum = lookup_clause_name(&normalized_name);
943    let (kind, data) = match clause_enum {
944        crate::parser::ClauseName::NumThreads => (0, ClauseData { default: 0 }),
945        crate::parser::ClauseName::If => (1, ClauseData { default: 0 }),
946        crate::parser::ClauseName::Private => (
947            2,
948            ClauseData {
949                variables: ptr::null_mut(),
950            },
951        ),
952        crate::parser::ClauseName::Shared => (
953            3,
954            ClauseData {
955                variables: ptr::null_mut(),
956            },
957        ),
958        crate::parser::ClauseName::Firstprivate => (
959            4,
960            ClauseData {
961                variables: ptr::null_mut(),
962            },
963        ),
964        crate::parser::ClauseName::Lastprivate => (
965            5,
966            ClauseData {
967                variables: ptr::null_mut(),
968            },
969        ),
970        crate::parser::ClauseName::Reduction => {
971            let operator = parse_reduction_operator(clause);
972            (
973                6,

975                    reduction: ManuallyDrop::new(ReductionData { operator }),
976                },

1002        // OpenACC-only clause names map to unknown in the OpenMP C API layer
1003        crate::parser::ClauseName::Copy
1004        | crate::parser::ClauseName::CopyIn
1005        | crate::parser::ClauseName::CopyOut

1008        | crate::parser::ClauseName::Async

1010        | crate::parser::ClauseName::NumGangs
1011        | crate::parser::ClauseName::NumWorkers
1012        | crate::parser::ClauseName::VectorLength
1013        | crate::parser::ClauseName::Gang
1014        | crate::parser::ClauseName::Worker
1015        | crate::parser::ClauseName::Vector
1016        | crate::parser::ClauseName::Seq
1017        | crate::parser::ClauseName::Independent
1018        | crate::parser::ClauseName::Auto
1019        | crate::parser::ClauseName::DeviceType
1020        | crate::parser::ClauseName::Bind
1021        | crate::parser::ClauseName::DefaultAsync
1022        | crate::parser::ClauseName::Link
1023        | crate::parser::ClauseName::NoCreate
1024        | crate::parser::ClauseName::NoHost

1052/// Parse reduction operator from clause arguments.
1053///
1054/// Extracts the operator from reduction clause like "reduction(+: sum)".
1055/// Returns integer code for the operator type.
1056///
1057/// ## Operator Codes:
1058/// - 0 = +  (addition)      - 5 = ^  (bitwise XOR)
1059/// - 1 = -  (subtraction)   - 6 = && (logical AND)
1060/// - 2 = *  (multiplication) - 7 = || (logical OR)
1061/// - 3 = &  (bitwise AND)   - 8 = min
1062/// - 4 = |  (bitwise OR)    - 9 = max
1063fn parse_reduction_operator(clause: &Clause) -> i32 {
1064    // Use the IR-level reduction operator parser to canonicalize the operator
1065    if let ClauseKind::Parenthesized(ref args) = clause.kind {
1066        let args = args.as_ref();
1067        // Find operator before top-level ':' using IR helper
1068        if let Some((op_str, _)) = split_once_top_level_colon(args) {
1069            let op_trim = op_str.trim();
1070            if let Ok(op_enum) = crate::ir::convert::parse_reduction_operator(op_trim) {
1071                use crate::ir::ReductionOperator;
1072                return match op_enum {
1073                    ReductionOperator::Add => 0,
1074                    ReductionOperator::Subtract => 1,
1075                    ReductionOperator::Multiply => 2,
1076                    ReductionOperator::BitwiseAnd => 3,
1077                    ReductionOperator::BitwiseOr => 4,
1078                    ReductionOperator::BitwiseXor => 5,
1079                    ReductionOperator::LogicalAnd => 6,
1080                    ReductionOperator::LogicalOr => 7,
1081                    ReductionOperator::Min => 8,
1082                    ReductionOperator::Max => 9,
1083                    _ => 0,
1084                };
1085            }
1086        }
1087    }
1088
1089    0 // Default to plus
1090}

1092/// Parse schedule kind from clause arguments.
1093///
1094/// Extracts schedule type from clause like "schedule(dynamic, 4)".
1095/// Returns integer code for the schedule policy.
1096///
1097/// ## Schedule Codes:
1098/// - 0 = static   (default, divide iterations evenly)
1099/// - 1 = dynamic  (distribute at runtime)
1100/// - 2 = guided   (decreasing chunk sizes)
1101/// - 3 = auto     (compiler decides)
1102/// - 4 = runtime  (OMP_SCHEDULE environment variable)
1103fn parse_schedule_kind(clause: &Clause) -> i32 {
1104    if let ClauseKind::Parenthesized(ref args) = clause.kind {
1105        let args = args.as_ref();
1106        // Use IR schedule parser to canonicalize modifiers and kind
1107        let config = ParserConfig::default();
1108        if let Ok(crate::ir::ClauseData::Schedule { kind, .. }) =
1109            crate::ir::convert::parse_schedule_clause(args, &config)
1110        {
1111            use crate::ir::ScheduleKind;
1112            return match kind {
1113                ScheduleKind::Static => 0,
1114                ScheduleKind::Dynamic => 1,
1115                ScheduleKind::Guided => 2,
1116                ScheduleKind::Auto => 3,
1117                ScheduleKind::Runtime => 4,
1118            };
1119        }
1120    }
1121    0 // Default to static
1122}

1246/// Free clause-specific data.
1247/// Free clause-specific data (internal helper).
1248///
1249/// Handles cleanup for union types where different variants need different
1250/// cleanup strategies. Currently only variable lists need explicit freeing.
1251///
1252/// ## Design Note:
1253/// Most clause data (schedule, reduction, default) are Copy types that don't
1254/// need cleanup. Only heap-allocated variable lists need explicit freeing.
1255///
1256/// ## IMPORTANT: ClauseData is a C union!
1257/// The ClauseData union has 4 fields, but only ONE is active per clause:
1258///   - `variables: *mut OmpStringList` - used by kinds 2-5 (private/shared/firstprivate/lastprivate)
1259///   - `reduction: ReductionData` - used by kind 6 (reduction operator only, NO variables pointer)
1260///   - `schedule: ScheduleData` - used by kind 7 (schedule policy only)
1261///   - `default: i32` - used by other kinds
1262///
1263/// Reduction clauses (kind 6) do NOT use the `variables` field. Trying to free
1264/// clause.data.variables on a reduction clause would read garbage memory from the
1265/// wrong union variant (the bytes of ReductionData::operator interpreted as a pointer).
1266fn free_clause_data(clause: &OmpClause) {
1267    unsafe {
1268        // Free variable lists if present
1269        // Clause kinds with variable lists (see convert_clause):
1270        //   2 = private, 3 = shared, 4 = firstprivate, 5 = lastprivate
1271        // Other kinds use different union fields:
1272        //   6 = reduction (uses .reduction field, NOT .variables)
1273        //   7 = schedule (uses .schedule field, NOT .variables)
1274        if clause.kind >= 2 && clause.kind <= 5 {
1275            let vars_ptr = clause.data.variables;
1276            if !vars_ptr.is_null() {
1277                roup_string_list_free(vars_ptr);
1278            }
1279        }
1280    }
1281}
