fn check_no_unknowns(mappings: &[(String, i32)], context: &str)
Expand description

Assign unique numeric placeholders for entries that were emitted as UNKNOWN_KIND. This prevents multiple ACC_* macros from expanding to the identical value (e.g. 999) which would cause duplicate case labels when used in C/C++ switch statements. Fail the build if any mapping entries are still UNKNOWN_KIND.

This enforces the invariant that the AST must exhaustively declare all ACC macros expected by the compatibility layer. When missing entries are detected the generator panics with a clear message listing the missing identifiers to make remediation straightforward for the developer.
